# 从 Figma 生成页面

你将根据 Figma 页面/Frame URL 生成完整的前端页面。

## 核心原则

目标是页面在浏览器中的视觉效果与设计稿一致。用最简洁的代码结构实现相同视觉效果。

## 第一步：检测项目技术栈

1. `package.json` → 框架、样式方案、路由库
2. `tsconfig.json` → 语言、路径别名
3. 扫描项目结构 → 页面目录、组件目录、路由方式
4. 读取已有页面 → 匹配写法模式

## 第二步：获取 Figma 设计数据

1. 调用 `get_node(depth: 10)` 获取页面完整视觉结构
2. 调用 `get_node_css(recursive: true)` 获取样式
3. 如有引用组件，调用 `get_component_variants`
4. 如有图标/矢量，调用 `export_svg`

## 第三步：盘点已有组件

扫描项目组件目录，将设计中的 UI 单元与已有组件匹配：

- 直接复用 → import 使用
- 扩展复用 → 修改已有组件
- 新建 → 才生成新组件

## 第四步：分析复杂度

### 简单页面（≤5 个区块）

直接生成页面 + 拆分组件。

### 复杂页面（>5 个区块）

1. 先实现完整 UI
2. 再拆分组件
3. 重构引用

### 组件拆分标准

- 重复 2 次以上 → 提取
- 有独立交互 → 提取
- 其余保持内联

## 第五步：生成代码

- 页面放到项目约定的页面目录
- 引用已有/新建的组件
- 预留事件接口（导航、数据加载、表单提交）
- 响应式布局遵循设计稿断点（如有）

## 第六步：验证

构建通过 + dev server 中视觉效果正确。

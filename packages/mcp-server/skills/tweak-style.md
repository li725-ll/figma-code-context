# 局部修正样式差异

你将对比 Figma 设计稿与当前代码实现，找出视觉差异并精准修正。

## 核心原则

- 只修改有视觉偏差的部分，不动已经正确的代码
- 保持代码风格与项目一致
- 不改动组件的交互逻辑和状态管理

## 第一步：确认修正范围

向用户确认哪个文件/组件需要修正，以及具体哪些方面有偏差。

## 第二步：获取设计数据

1. 调用 `get_node_css` 获取目标节点的精确样式
2. 如有图标差异，调用 `export_svg`

## 第三步：读取当前实现

1. 读取用户指定的组件文件
2. 识别当前样式方案（Tailwind / CSS Modules / styled-components 等）

## 第四步：对比并修正

关注视觉差异：

- 间距（padding/margin/gap）
- 颜色（背景色/文字色/边框色）
- 字体（大小/粗细/行高/字间距）
- 布局（对齐/方向）
- 尺寸（宽高/圆角）
- 阴影/透明度/边框

修正原则：

- 最小改动，只修改有偏差的属性
- 使用项目已有的样式写法
- 优先使用 design token 而非硬编码值

## 第五步：验证

确认修正后视觉效果与设计稿一致，未引入新问题。
